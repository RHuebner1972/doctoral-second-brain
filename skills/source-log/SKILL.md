# Source Log

## Purpose
Build and maintain an organized, annotated log of every source the learner has read — so sources are retrievable, ranked by relevance, and ready to synthesize.

## The Problem With Unorganized Sources
Most doctoral learners have read more than they think — but can't retrieve what they've read when they need it. Sources are scattered across PDF folders, browser bookmarks, Zotero libraries, and memory. The source log creates one canonical list.

## Source Log Entry Format

Each source entry contains:

```
CITATION: [Full APA 7th reference, formatted and ready to paste]
TYPE: [Journal article / Book / Dissertation / Report / Other]
CORE CLAIM: [One sentence — what the author argues or found]
METHOD: [Research design and sample, briefly]
KEY FINDING: [The most important thing this source establishes]
KEY QUOTE: "[Exact quote]" (p. X)
THEMES: [#theme1 #theme2 — tag with dissertation chapter themes]
RELEVANCE: [High / Medium / Low — how directly does this inform the dissertation?]
USED IN: [Chapter 2, Theme 1 / Chapter 3, Methodology Justification / etc.]
NOTES: [Anything else worth remembering — limitations, connections to other sources]
```

## Relevance Rating Guide

| Rating | Meaning |
|--------|---------|
| **High** | Directly addresses the core phenomenon, population, context, or methodology; will be cited in the dissertation |
| **Medium** | Addresses a related construct or provides useful background; may be cited; useful for synthesis |
| **Low** | Tangentially related; provides context or validates a minor point; probably won't be cited |

## Source Types and How to Handle Each

**Empirical studies (primary sources):**
- Record method and sample
- Note specific findings (not just "they studied X")
- Rate relevance to each theme

**Theoretical/conceptual articles:**
- Record the theory's propositions
- Note key theorists cited (may need to go find the originals)
- Flag for theoretical framework section

**Dissertations:**
- Mine for their literature reviews and reference lists
- Note their gap — often similar to yours and a good validation
- Note their methodology for precedent

**Books:**
- Record chapter-level; don't log entire books as single entries
- Note page ranges for key arguments

**Reports / Gray literature:**
- Validate practitioner relevance
- Note if peer-reviewed or not (important for scholarly credibility)

## Source Log Master Table (Simplified View)

For a high-level overview, maintain a summary table:

| Author (Year) | Topic | Method | Relevance | Theme | Status |
|--------------|-------|--------|-----------|-------|--------|
| Smith (2022) | Servant leadership + faculty | Survey | High | Theme 1, Theme 2 | Logged |
| Jones (2021) | Organizational commitment | Interview | High | Theme 2 | Logged |
| Brown (2019) | Leadership theory review | Lit review | Medium | Framework | Logged |

## Recommended Tools
- **Zotero** — free, integrates with Word; store full entries + PDFs + notes
- **Mendeley** — similar to Zotero; good for annotation
- **Notion database** — flexible, taggable, linkable
- **Excel/Sheets** — low-tech but effective for the summary table
- **Obsidian** — best for linking notes and building the second brain

## Process

### Step 1 — Add a New Source
When the learner provides a source (citation, abstract, or notes):
1. Extract all required fields
2. Write the core claim (one sentence)
3. Assign relevance and themes
4. Format the APA reference

### Step 2 — Weekly Source Review
Once per week:
- Are any high-relevance sources not yet connected to a chapter section?
- Are any sources duplicated?
- Are there any gaps in coverage (themes with fewer than 5 high-relevance sources)?

### Step 3 — Pre-Writing Source Pull
Before drafting any section:
- Filter the log by theme and relevance
- Extract all High sources for that theme
- Use the key claims and quotes to build the synthesis

## Output Format
1. **Formatted source entry** (all fields filled)
2. **APA reference** (ready to paste)
3. **Summary table row** (for the master table)
4. **Theme tags** (connected to dissertation structure)
5. **Coverage check** (if multiple sources provided — are themes adequately covered?)
