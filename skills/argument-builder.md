# Argument Builder

## Overview

A dissertation is, at its core, an extended argument. Every section makes claims, supports them with evidence, and draws conclusions. Weak arguments aren't always weak because of the evidence — they're weak because the *structure* is unclear: claims that aren't stated, evidence that doesn't connect, or conclusions that go beyond what the evidence supports. This skill diagnoses argument structure problems and rebuilds them.

## The Anatomy of a Scholarly Argument

Every scholarly argument — at every level from a paragraph to a full chapter — has these components:

```
Claim → Evidence → Warrant → Conclusion
```

| Component | What It Is | Example |
|---|---|---|
| **Claim** | The assertion you're making | "Servant leadership is positively associated with employee retention." |
| **Evidence** | The data, studies, or facts that support it | "(Liden et al., 2008; van Dierendonck, 2011; Ehrhart, 2004)" |
| **Warrant** | The logical link between evidence and claim | "These studies consistently found that when leaders prioritize follower needs, followers develop organizational commitment and intend to remain." |
| **Conclusion** | What follows from claim + evidence | "Therefore, examining servant leadership behaviors in community college settings is warranted." |

**Most weak arguments are missing the warrant** — the author states a claim, drops citations, then jumps to a conclusion without explaining the connection.

## Argument Diagnosis Framework

When evaluating an argument, check for these five failure modes:

### 1. Missing Claim (No Thesis)
The writing describes without arguing. There's no sentence that states *what you want the reader to believe.*

**Symptom:** Section reads like a summary rather than an argument.
**Fix:** Add an explicit topic sentence that makes a claim: "The evidence suggests that X because Y."

### 2. Unsupported Claim
A claim is made without evidence.

**Symptom:** "Servant leadership is effective for faculty retention." — no citation.
**Fix:** Add empirical support or acknowledge it as a theoretical proposition.

### 3. Missing Warrant (Logical Gap)
Evidence is present, but the connection to the claim isn't explained. Reader must infer the logic.

**Symptom:** Claim. Citations. Conclusion. No explanation of *why* the evidence supports the claim.
**Fix:** Add 1–2 sentences explicitly linking the evidence to the claim.

### 4. Overclaiming
The conclusion goes beyond what the evidence supports.

**Symptom:** "These studies prove that servant leadership is the best approach for all organizations."
**Fix:** Hedge the conclusion to match the scope of the evidence: "These findings suggest that servant leadership may be effective in non-profit educational settings."

### 5. Circular Argument
The claim and conclusion are the same thing stated differently.

**Symptom:** "Servant leadership improves retention because leaders who focus on followers help them stay."
**Fix:** Introduce a mechanism or theoretical explanation: *why* does this relationship exist?

## Argument Structure Patterns for Dissertation Writing

### Pattern 1: Support (Most Common)
Use when you want to establish a claim as supported by the literature.
```
[Claim] → [Evidence 1] → [Evidence 2] → [Warrant] → [Conclusion]
```
*"Servant leadership is associated with follower well-being and retention (Evidence 1, 2, 3).
These studies consistently demonstrate that [warrant — mechanism]. This pattern supports
the contention that [conclusion]."*

### Pattern 2: Contrast and Resolve
Use when the literature is divided; you're arguing for a position.
```
[Claim A exists] → [Counter-claim B also exists] → [Reason A is more applicable here] → [Resolution]
```
*"While Smith (2018) found X, Jones (2020) found Y. However, Jones's study was limited to
[context], which differs significantly from [your context]. In settings characterized by
[your setting], X is the more applicable finding because [reason]."*

### Pattern 3: Gap → Need → Study
Use in problem statement, purpose statement, and introduction.
```
[What is known] → [What is not known] → [Why it matters] → [Proposed study as response]
```

### Pattern 4: Framework → Prediction → Test
Use in quantitative studies linking theory to hypotheses.
```
[Theory proposes X] → [This predicts that Y will occur under condition Z] → [This study tests whether Y occurs]
```

## Paragraph-Level Argument Checklist

For each paragraph, confirm:
- [ ] It opens with a topic sentence that makes a claim (not a statement of intent)
- [ ] Evidence follows the claim (not before)
- [ ] A warrant or analytical sentence connects evidence to the claim
- [ ] The paragraph closes by advancing the argument (not just summarizing)
- [ ] It connects logically to the next paragraph via a transition

---

## Section-Level Coherence Check

Ask these questions of each major section:
1. **What is the main claim of this section?** (Can you state it in one sentence?)
2. **Does each paragraph contribute to that claim, or does it wander?**
3. **Is there a logical sequence** — does each paragraph build on the previous one?
4. **Does the section conclude** — does it land somewhere, or just stop?
5. **Does the conclusion connect to the next section?** (Transition)

---

## Rebuilding a Weak Argument

If an argument is structurally broken, use this sequence:

1. **State the claim explicitly** — write it as a single sentence
2. **List the evidence** — what sources support it?
3. **Write the warrant** — why does that evidence support that claim?
4. **State the conclusion** — what follows?
5. **Check the conclusion against the claim** — they should be related but not identical

Then draft the paragraph using this structure.
