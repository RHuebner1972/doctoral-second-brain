# Qualitative Data Analysis

## Purpose
Systematically analyze qualitative data using rigorous, transparent procedures that transform raw interview transcripts, observation notes, and documents into meaningful themes and theoretical insights.

## Process

### Step 1 — Prepare Your Data

**Data Organization**

- **Transcription**
  - Interview audio should be transcribed verbatim (every word, including filler words like "um", "uh")
  - Use a professional transcription service (more accurate) or transcribe yourself (slower but intimate with data)
  - Include timestamps if helpful for long interviews
  - De-identify during transcription (remove participant names; replace with ID codes)

- **File Management**
  - Organize files by participant (P1_Interview_1.txt, P1_Interview_2.txt, etc.)
  - Keep backup copies
  - Create a master file documenting participant demographics & data collected

- **Data Immersion**
  - Listen to audio OR read transcripts multiple times
  - Take notes on initial impressions, patterns, or puzzles
  - Don't jump to analysis; sit with the data first

**Quality Check on Data**
- Verify transcripts for accuracy (re-listen to sections; check for misheard words)
- Review for confidentiality (replace names, specific locations, identifying details)

---

### Step 2 — Choose Your Coding Approach

**Common Qualitative Analysis Methods**

| Method | Best For | Process |
|--------|----------|---------|
| **Thematic Analysis** | Identifying patterns/themes across data | Open coding → candidate themes → refine themes |
| **Grounded Theory** | Developing theory from data | Open coding → axial coding → selective coding → core category |
| **Phenomenology** | Understanding lived experience | Focus on descriptions; reduce to essence; identify meaning |
| **Narrative Analysis** | Analyzing stories & how people make meaning | Examine plots, characters, context; focus on narrative structure |
| **Ethnography** | Understanding cultural practices | Immersion; thick description; cultural interpretation |
| **Content Analysis** | Quantifying themes in text | Code data; count frequencies; describe patterns |

**Thematic Analysis (Most Flexible; Recommended for Novices)**

Braun and Clarke's (2006) six-phase approach:

1. **Familiarization** with data
2. **Open coding** (initial codes)
3. **Candidate themes** (organizing codes into themes)
4. **Review themes** against data
5. **Define & name themes**
6. **Write up findings**

---

### Step 3 — Conduct Open Coding

**What Is Coding?**
Assigning labels to chunks of text that represent meaningful units. Codes should capture *what the data is about* and *what is important about it*.

**Open Coding Process**

1. **Read a transcript** (or observation notes) line-by-line
2. **Identify meaningful chunks** (can be a word, phrase, sentence, or paragraph)
3. **Assign a code** (a label that captures the meaning)
4. **Move through entire transcript** systematically
5. **Repeat with next transcript** (you may create new codes or apply existing ones)

**Example**

| Original Text | Possible Codes |
|---------------|----------------|
| "I've been teaching for 15 years, and I've seen curriculum change dramatically. It's exciting but also exhausting trying to keep up." | Teacher identity; Career longevity; Curriculum innovation; Emotional labor; Change fatigue |
| "I use a mix of lecture and small group work. It keeps students engaged because they're doing something, not just listening." | Teaching strategy; Student engagement; Active learning; Instructional variation |

**Coding Styles**

- **In Vivo Codes**: Use participants' own words
  - Participant: "I feel like I'm drowning in emails"
  - In vivo code: "Drowning in emails" (or shorten: "Email overload")

- **Conceptual Codes**: Researcher-generated labels capturing broader meaning
  - Participant: "My principal saw my low test scores and immediately blamed my instruction"
  - Conceptual code: "Attribution of blame to teachers" or "Accountability pressure"

- **Descriptive Codes**: Straightforward labels
  - Code: "Concerns about student mental health"

**Tips for Open Coding**
- Start coding 2 transcripts thoroughly to develop a preliminary codebook
- Be generous; create many codes initially (you'll consolidate later)
- Code the same data differently if multiple meanings fit
- Keep a codebook: list each code + definition + example
- Note interesting patterns or questions in margin

### Step 4 — Develop a Codebook

**Codebook Template**

```
Code Name: Teacher Isolation

Definition: Feeling separated from colleagues; lack of collaboration or 
support; working in professional silos.

When to Use: When participants express loneliness, lack of collegial 
support, or inability to share teaching challenges with peers.

When NOT to Use: When participants discuss work-life balance broadly or 
taking breaks alone (which is by choice).

Example from Data:
P3: "I'm the only one on my grade level teaching this way. There's nobody 
to bounce ideas off of. It's just me in my classroom."

Notes: Related to codes: "Lack of PLC," "Professional community," 
"Teaching autonomy"
```

**Codebook Best Practices**
- Include 5–10 codes in preliminary codebook; expand as needed
- Review & refine codebook after coding 2–3 transcripts
- Ensure codes are sufficiently distinct (don't create overlapping codes)
- Share codebook with research team for consistency (if collaborative)

### Step 5 — Organize Codes Into Candidate Themes

**What Is a Theme?**
A broader pattern that brings together multiple, related codes. Themes represent core ideas in your data.

**Process**

1. **List all codes** you've developed
2. **Look for relationships**
   - Which codes seem related?
   - Which capture similar meaning?
   - Are some codes subcategories of larger patterns?

3. **Group codes into candidate themes**
   - Example codes: "Time management struggles," "Grading load," "Planning pressure," "Meeting fatigue"
   - Candidate theme: "Administrative & Workload Burden"

4. **Name themes clearly**
   - Use a phrase that encapsulates the theme
   - Make themes conceptually distinct from one another

**Example Theme Map**

```
Candidate Theme 1: "Support Systems"
├─ Colleague collaboration
├─ Mentorship relationships
├─ Administrative support
└─ Professional development access

Candidate Theme 2: "Barriers to Efficacy"
├─ Student diversity (unpreparedness)
├─ Inadequate resources
├─ Standardized testing pressure
└─ Limited autonomy

Candidate Theme 3: "Personal Impact"
├─ Emotional exhaustion
├─ Self-doubt/imposter syndrome
├─ Passion for teaching
└─ Burnout risk
```

---

### Step 6 — Review & Refine Themes

**Validity Check**

1. **Does each theme have sufficient data?**
   - If a theme has only 1 mention, it may not be significant; consider eliminating
   - Strong themes typically have multiple supporting quotes across several participants

2. **Are themes distinct?**
   - Do two themes overlap heavily? If so, consider merging
   - Are boundaries between themes clear?

3. **Do all data fit into themes?**
   - Are there codes/data orphaned (not fitting any theme)?
   - Reorganize or create new themes as needed

4. **Member Checking** (optional but valuable)
   - Share emerging themes with 2–3 participants
   - Ask: "Do these themes capture your experience? Is anything missing?"
   - Incorporate feedback

5. **Team Review** (if collaborative)
   - Have co-coders review theme organization
   - Discuss any divergences; reach consensus

### Step 7 — Define & Name Themes

**For Each Theme, Document**

| Element | Description |
|---------|-------------|
| **Theme Name** | Clear, evocative, conceptually accurate |
| **Definition** | 2–3 sentences explaining the theme |
| **Subthemes** (if any) | Smaller patterns nested within |
| **Supporting Codes** | List of codes included in this theme |
| **Representative Quotes** | 2–3 exemplary quotes from data |

**Example Theme Definition**

```
Theme: Professional Isolation & Fragmentation
Definition: Teachers experience limited opportunities for meaningful 
collaboration with colleagues. They work in relative silos, have few 
spaces to discuss challenges, and lack structures for collective problem-solving. 
This isolation contributes to feelings of vulnerability and reduces 
opportunities for shared learning and support.

Subthemes:
- Lack of collaboration structures (PLCs, grade-level teams)
- Emotional distance from colleagues
- Absence of collegial mentorship
- One-way feedback from administration (vs. peer dialogue)

Supporting Codes:
- Colleague collaboration
- Collegial relationships
- PLC engagement (or lack thereof)
- Administrative feedback vs. peer feedback

Representative Quotes:
P2: "I feel like I'm teaching in a vacuum. My door is closed; 
I don't see what others are doing, and they don't see what I'm doing."

P7: "There's no time to talk to other teachers. We pass in the hallways, 
but real conversation? That doesn't happen."
```

### Step 8 — Memo-Writing for Conceptual Development

**What Are Memos?**
Reflective writing during analysis to capture emerging insights, connections, and theoretical thinking. Memos are for YOU; they're not part of the final dissertation, but they drive your interpretation.

**Types of Memos**

- **Descriptive Memos**: What is this code/theme about?
  ```
  Memo (Code: Teacher Efficacy):
  Participants reference their sense of effectiveness in the classroom—or 
  lack thereof. Some feel highly efficacious (I can impact student learning), 
  while others question their impact (Am I making a difference?). Efficacy 
  seems tied to student behavior, test scores, and administrative feedback. 
  Worth exploring: Does low efficacy lead to burnout, or does burnout 
  undermine efficacy? Likely bidirectional.
  ```

- **Analytical Memos**: How are codes/themes connected? What patterns emerge?
  ```
  Memo (Themes: Support Systems × Efficacy):
  Noticing a pattern: Participants with strong collegial support (shared 
  planning time, co-teaching, mentoring relationships) report higher efficacy. 
  Conversely, isolated teachers doubt their effectiveness more. This suggests 
  that professional community may be protective against efficacy loss. Could 
  this be a core finding?
  ```

- **Theoretical Memos**: How do themes relate to theory? What's emerging conceptually?
  ```
  Memo (Theory Connection: Self-Efficacy & Social Support):
  Bandura's social cognitive theory emphasizes that self-efficacy comes from 
  mastery, vicarious experience, social persuasion, and physiological states. 
  In this data, teachers build efficacy through peer mentoring (vicarious 
  experience) and collegial problem-solving (social persuasion). This 
  framework helps explain WHY professional community matters.
  ```

**Memo-Writing Practice**
- Write memos throughout analysis, not just at the end
- Keep memos in a separate document
- Date memos to track your thinking over time
- Review memos when writing up findings; they often become seeds for discussion

### Step 9 — Reflexivity & Positionality

**Reflexivity = Self-Awareness During Analysis**

Acknowledge how your biases, assumptions, and experiences shape interpretation.

**Reflexivity Journal Prompts**

- What assumptions did I bring to this interview/observation?
- Did my background influence how I heard this participant's story?
- What surprised me? What challenged my expectations?
- Am I drawn to certain codes/themes because of my biases?
- What am I *not* seeing or hearing?
- How might another researcher code this differently?

**Example Reflexivity Entry**

```
Reflexivity Journal – Date: 3/15/2024
Data: Interview with P4 (teacher with 2 years of experience)

Assumption I brought: New teachers are typically enthusiastic. I expected 
high motivation and energy.

What I heard: P4 is actually quite discouraged. She loves teaching but feels 
unsupported by her school.

What challenged me: I realized I was making assumptions about novices. My 
own first years were different (I had a mentor; P4 doesn't). I need to be 
careful not to impose my experience onto her data.

Bias check: Am I over-coding things related to "lack of mentorship" because 
I value mentorship? Probably. I should ensure I'm not forcing this theme 
artificially. Let the data speak.
```

### Step 10 — Write Findings With Data

**Findings Structure**

Typically: **Theme + Explanation + Supporting Quotes**

**Example Finding Write-Up**

```
Thematic Findings

Theme 1: Professional Isolation & Fragmentation
Teachers consistently described working in relative isolation. This isolation 
occurred at two levels: (a) structural (school design prevents collaboration) 
and (b) emotional (teachers feel unseen and unsupported by colleagues). 

All 10 participants mentioned limited contact with colleagues about teaching 
practice. One teacher explained: "I feel like I'm teaching in a vacuum. My door 
is closed; I don't see what others are doing, and they don't see what I'm doing" 
(P2). Another noted the emotional toll: "There's no time to talk to other 
teachers. We pass in the hallways, but real conversation? That doesn't happen" 
(P7).

The absence of collaborative structures (professional learning communities, 
grade-level planning teams) meant teachers had few spaces to discuss instructional 
strategies or challenges. One teacher reflected: "I wish I could bounce ideas 
off someone. Instead, I figure things out alone and hope I'm doing it right" (P5).

This theme aligns with existing literature on teacher burnout (Goddard et al., 
2006), which identifies collegial support as a protective factor. In this sample, 
its absence appeared to amplify stress and reduce instructional innovation.
```

**Best Practices in Findings**
- Use quotes liberally; let participants' voices be heard
- Contextualize quotes (who said it, in what context?)
- Weave quotes into your analytical narrative (don't just paste and move on)
- Balance multiple perspectives (show variation; don't over-represent one voice)
- Link themes to relevant literature (even in Findings)

## Qualitative Analysis Checklist

- [ ] Transcribe or organize all raw data
- [ ] Create a codebook with clear code definitions
- [ ] Conduct open coding on first 2–3 data sources
- [ ] Apply codes systematically to all data
- [ ] List all codes and review for redundancy/clarity
- [ ] Group codes into candidate themes
- [ ] Verify themes have sufficient data support
- [ ] Conduct member checking (if feasible)
- [ ] Write memo notes throughout analysis
- [ ] Track reflexivity and positionality
- [ ] Refine theme definitions & names
- [ ] Select representative quotes for each theme
- [ ] Prepare findings with balanced perspective
- [ ] Align findings with research questions

---

## Quality Indicators for Qualitative Analysis

✓ Codes are clearly defined and consistently applied  
✓ Themes are distinct, well-supported, and named clearly  
✓ Findings include multiple perspectives (not just dominant voice)  
✓ Raw data (quotes) directly supports interpretations  
✓ Researcher reflexivity is documented  
✓ Limitations of analysis are acknowledged  
✓ Findings address original research questions  
✓ Sufficient data density (themes not thin or under-supported)
